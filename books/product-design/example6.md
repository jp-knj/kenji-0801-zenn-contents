あなたはプロダクトデザイナーとしての視点から、ステークホルダーのマネジメントについての知識を深めることの重要性を理解しているでしょう。この技術記事では、その方法と具体的な手法について解説します。デザイナーやフロントエンドエンジニアの皆さんにとって、役立つ情報を提供することを目的としています。

## 共同作業 
ステークホルダーのマネジメントとは、あなたが関与するプロジェクトに関わる他の人々との関係を管理することを指します。これには、彼らの期待を管理し、適切な情報をタイムリーに更新することが含まれます。ステークホルダーを適切に管理することで、プロジェクトは前進し、目標を達成することができます。

ステークホルダーのマネジメントを知ることは以下のような利点があります。
- キャリアが進むにつれてプロジェクトをリードする能力を高める。
- プロジェクトのダイナミクスをより理解し、いつ、どのように、そして何をパートナーとコミュニケートするかを知ることで、関与するプロジェクトでの協力スキルを向上させる。

特定のプロジェクトをリードしていない場合でも、以下のフレームワークを使用して、継続的に連携している人々との関係を向上させることができます。そのためには、フレームワークに従い、ステップ3、5、6をスキップしてください。

## ステークホルダー管理テーブルの作成
あなたが組織内でデザインシステムの実装をリードしているとしましょう。それを実現するためには、複数の当事者の同意を得る必要があります。関与するすべての当事者を効果的に管理し、望ましい結果を得るために、次のステップに従ったスプレッドシートを作成してください。

### ステークホルダー管理の手法
以下の手法は、ステークホルダーのマネジメントを効果的に行うためのものです。

#### 1. ステークホルダーのリストアップ
プロジェクトに関与するすべての人々や団体をリストアップします。具体的な人物や団体（例：デザインチーム、エンジニアリング、プレス、クライアントなど）を考えてください。

#### 2. 優先順位の決定
ステークホルダーを優先順に並べ、彼らとのコミュニケーションにどれだけの労力と時間を投資する必要があるかを見極めます。

図14.2 ステークホルダー管理の優先順位。

- **密接に管理する**：プロジェクトの主要なプレイヤー。
- **満足させ続ける**：プロジェクトには直接関与していないが、影響力がある人々。
- **情報を提供し続ける**：プロジェクトに戦術的または実装レベルで影響を与える人々。
- **モニタリングする**：プロジェクトのこの段階で主要でない人々。

#### 3. 彼らの感情を示す
各ステークホルダーの現在と望ましい感情を、ポジティブ、ニュートラル、ネガティブのいずれかで追跡します。

#### 4. 彼らの関心を明確にする
各ステークホルダーが何を重視しているのかを明確にします。

#### 5. このプロジェクトが彼らの関心をどのように満たすかを示唆する
これはステークホルダーに影響を与えるための鍵となる洞察です。

#### 6. エンゲージメント戦略の開発
各ステークホルダーの優先順位と、プロジェクトに対する彼らの現在の感情との間のギャップに基づいて、どのように彼らをプロジェクトに関与させ、更新し続けるかを決定します。

**例**
- スタンダップでの毎日の更新。
- 2週間ごとのスプリントレビュー会議。
- 週に1回のメールでの更新。
- Slackグループ#design-systemでの週次更新。
- 金曜日の11:00に30分のレビュー。
- デザイン批評中の週次更新。
- ユーザビリティテストセッションへの招待。
- 月に1回のメールでの更新。

### 他者を説得する: デザイナーの新たなスキルセット
#### 1. はじめに
デザイナーが初めての仕事を始めるまで学び始めることの一つは、非デザイナーとの共同作業です。学習中、デザイナーは通常、他のデザイナーとプレゼンテーションを行ったり、協力したり、フィードバックを提供したりします。しかし、組織で働く現実は、デザイナーがデザインの背景がない人々と常に協力する必要があることです。

#### 2. 説得のアプローチ
あるエンジニアからのメッセージを想像してみてください：

「サム、フロントエンドでコードのリファクタリングを行っているので、デザイン上何も壊れていないか確認するセッションに参加してもらえますか？」

対して

「サム、私たちはコードのリファクタリングを行っており、これは将来のデザインシステムの統合を容易にするための一部です。デザインが最初に設計された通りにピクセルパーフェクトに保たれ、ユーザーがアプリ全体で一貫した経験を得ることを確認するために、セッションに参加してもらえますか？」

どちらが助けを求める意欲を高めますか？

#### 3. さまざまな役職へのアプローチ
以下は、異なる組織の役割にとって何が重要であるかの例です：

| ステークホルダーグループ | 重視するポイント |
| :---------------------- | :-------------- |
| 製品マネージャー         | ユーザーのニーズの満足、定時出荷、ロードマップとの一致など |
| 製品の責任者             | 戦略的な整合性、製品ロードマップなど |
| エンジニア               | 簡潔で一貫性のあるコードベース、スケーラビリティなど |
| エンジニアリングの責任者  | チームのパフォーマンス、システムアーキテクチャなど |
| ...                    | ...            |

次回、PMからの承認が必要な場合、彼らに「この機能を定時に出荷して、ロードマップの通りに進行するために、週末までにそれを必要としています」と伝えることができます。

#### 4. 効果的なコミュニケーション
あなたのポイントをさらに説得力のあるものにするためには、「デザインのコミュニケーション」の章の「あなたの推論を明確にする」という技法と、相手の利益を組み合わせることができます。

デザイナーとして、他者との効果的なコミュニケーションと説得は、日々の業務の中での成功の鍵となります。

### ステークホルダーのマネジメント: 人間関係の構築と他者を助ける

#### 1. **人間関係の価値**
組織内の他の人々との接触を持つことは、常勤雇用の主な特典だと考えています。これらの関係を強化することは、現在の組織での成功につながりますが、長期的にはさらに影響があるかもしれません。これらの人々は、あなたの未来のキャリアや将来の共同創業者、顧客、ベンダー、投資家、または同僚としての決定的な瞬間の主要なプレイヤーである可能性があります。

#### 2. **人間関係の築き方**
私には完璧な人間関係の築き方はありませんが、私が試した方法は人々に役立つことです。特にキャリアの初めの頃は、要求に「はい」と答える傾向があります。それがどのようにあなたに利益をもたらすかはすぐには明らかでないかもしれませんが、後で明らかになるかもしれません。

#### 3. **デザイナーとしての貢献**
デザイナーとして、組織内のほぼすべての機能と協力する機会があるかもしれません。オフィスマネージャーが注文する会社のロゴのTシャツデザインにアドバイスが必要であったり、セールスチームがプレゼンテーションのサポートを必要としている場合、エンジニアリング部門が主催するミートアップのポスターに助けが必要な場合など、さまざまなリクエストに対応することで、以下の利点が得られます：

- あなたの影響範囲を拡大し、より野心的なプロジェクトを実装するのを助けます。
- その役職に関する更なる理解を得ることができます。
- あなたは彼らの作業の仕方を観察することで、繰り返される問題を認識することができます。
- 組織の運営方法に関して広い視野を持つことができます。

#### 4. **外部の組織でのアプローチ**
2013年、私はテルアビブのローカルハッカソンに参加しました。私は前に会ったことのない人々のチームに参加し、作業を始めました。多くのデザイナーは関与しておらず、私たちの隣のチームは彼らの製品を見栄えの良いものにするのに明らかに苦労していました。技術的には彼らは私たちの競争相手でしたが、私は基本的なデザインの提案で彼らを助けました。私のチームのピッチは失敗でしたが、私が助けたチームは第一位を獲得しました。彼らは私をチームに招待し、チケットも確保してくれました。1年後、私は仕事を辞め、新しい会社の共同創業者として彼の一人に参加しました。
